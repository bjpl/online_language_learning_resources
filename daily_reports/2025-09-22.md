**Rationale**:
- Started with 5 languages (simple grid)
- Grew to 13 languages in one day
- Needed flexible system for future expansion
- Prevent layout breaking at different language counts

**Evolution**:
- 2-column grid (initial)
- 3-column grid (10 languages)
- 6-column grid (11+ languages)
- Scalable system (Vietnamese commit)

### Data Structure Consistency
**Decision**: Enforce strict data structure standards for all languages
**Rationale**:
- Finnish initially had wrong path and structure
- Required 3 fix commits to correct
- Established templates prevent future issues
- Consistent structure = reliable rendering

**Enforcement**:

### Language Filter UI
**Decision**: Two-row grid layout for language filters
**Rationale**:
- Single row too wide for 10+ languages
- Mobile responsiveness concerns
- Better visual balance
- Easier to scan and select

**Implementation**:
- CSS grid with auto-fit
- Responsive columns (2-6 based on screen size)
- Maintains usability at any language count
